# FTP 服务器辅助考试系统 — 项目介绍

## 1. 项目概述

本项目是一套面向教学与机房场景的 **FTP 文件服务 + Web 管理 + 在线考试辅助** 一体化系统。它在同一进程中协调运行：

- **FTP 服务**：供学生或考生使用标准 FTP 客户端上传、下载作业或答卷目录中的文件。
- **Web 统一门户**：提供入口导航、登录与权限分流。
- **管理员控制台**：监控连接与磁盘、管理 FTP 用户、查看日志、控制 FTP 进程、配置限速与连接数等。
- **用户文件浏览器**：在浏览器中完成列表、上传、下载、删除、重命名、建目录等操作（权限与用户主目录绑定）。
- **考试辅助模块**：支持试卷池管理、按场考试随机分卷、批量创建考生账号、导出账号信息、清理考试数据等（与 `ftp_root` 下的学生目录及试卷池目录配合使用）。

项目定位是 **局域网或机房内** 的快速部署方案：单命令启动 FTP 与 Web，数据库使用 **本地 MySQL**（见下文「运行前提」）。

## 2. 功能详解

以下按 **用户能接触到的界面** 与 **底层 FTP 能力** 分别说明，便于写说明书或答辩材料。

### 2.1 统一门户（`/`）

- **入口分流**：首页以卡片形式引导至「管理员控制台」与「用户文件浏览器」，并展示服务状态文案。
- **适用场景**：机房内张贴门户地址，学生进浏览器、管理员进后台，路径清晰。

### 2.2 登录与认证（`/login`、文件浏览器内嵌登录）

- **管理员**：通过统一登录页进入 `/admin`，会话与 **管理员作用域 Cookie** 绑定，避免与文件浏览器互相顶掉登录。
- **普通用户 / 考生**：文件浏览器（`/explorer`）自带登录页，支持 **记住密码、自动登录**（本地偏好）、用户名 **datalist** 历史提示。
- **技术要点**：后端支持 `X-Auth-Scope`（`admin` / `explorer`）、`auth_session` 参数、签名 Token 等多种方式，适合嵌入链接或调试。

### 2.3 管理员控制台（`/admin`，`admin_panel.html`）

左侧为 **五大模块**，右侧为对应内容区。

#### 2.3.1 仪表盘

- **核心指标卡片**：当前 **活跃 FTP 连接数**、**用户总数**、**数据传输量**（汇总展示）、**服务运行时间**。
- **图表**：**连接趋势**（Chart.js）、**磁盘使用情况**（饼图/环形类展示）。
- **最近活动**：基于日志或概览接口的 **实时刷新** 活动列表。
- **在线用户**：展示当前 FTP 在线用户列表及人数角标。
- **磁盘使用 TOP5**：在启用配额的前提下，突出 **配额使用率最高** 的用户，便于清理或调配额。

#### 2.3.2 用户管理

- **用户表格**：用户名、角色、主目录、**FTP 权限字**、配额、**限速**、启用状态、操作列。
- **添加用户（弹窗）**：用户名/密码、主目录（默认 `./ftp_root/` 前缀可改）、**权限预设**：
  - `elr`：只读（列表、读、下载类能力，具体以服务端解析为准）
  - `elradfmw`：完全访问
  - `elradfmwMT`：完全访问并含目录相关扩展权限
- **存储配额**：按 MB 配置，`0` 表示不限制。
- **速度限制**：按 KB/s 配置 FTP 传输上限，`0` 表示不限制。
- **编辑 / 删除**：对已有账号维护生命周期（与数据库 `users` 表一致）。

#### 2.3.3 服务器配置

- **基本参数**：监听地址、端口、最大连接数、超时（秒）、**FTP 欢迎标语**。
- **服务控制**：**启动 / 停止 / 重启** FTP 进程，状态指示灯与文案联动。
- **保存配置**：将界面参数提交到后端，与 `ftp_config` 等持久化逻辑配合（实际生效范围以实现为准）。

#### 2.3.4 操作日志

- **终端风格视图**：深色背景、等宽字体，接近「控制台日志」体验。
- **筛选**：按用户名关键词过滤（前端输入框 + 刷新）。
- **实时流**：进入该页时可建立 **SSE/EventSource** 订阅日志流；离开页面自动关闭连接，节省资源。

#### 2.3.5 考试管理（嵌入管理台的一套完整流程）

与 `/exam_admin` 逻辑同源，集中在侧栏「考试管理」页，实现 **命题与机房运维** 的闭环：

| 功能块 | 说明 |
|--------|------|
| **创建考生账号** | 填写 **考试 ID**（字母数字下划线连字符）、**考生数量**（1–500）、**每人存储配额 MB**（0=不限）。一键生成后写入数据库与 FTP 用户体系，并准备学生目录。 |
| **上传试卷到池** | 选择考试、**点击或拖拽** 上传；支持 **多文件**；格式含 **ZIP / PDF / Word / TXT**；**ZIP 可自动解压** 入试卷池。 |
| **试卷池管理** | 列出当前考试下试卷文件，支持 **刷新**、**清空** 池（慎用）。 |
| **随机分卷** | 从该考试试卷池中 **随机抽取一份** 分发给每位尚未分卷的考生（实现为「每人一卷、卷面可不同」）。 |
| **一键清场** | 选择考试后删除 **该场所有考生账号及其文件**；界面有 **不可恢复** 警示。 |
| **考生列表（右侧大表）** | 展示用户名、密码、主目录、配额、已用量、状态；支持 **导出 CSV**、刷新、**分页**（上一页/下一页）；无数据时有空状态提示。 |

### 2.4 独立考试管理后台（`/exam_admin`，`exam_admin.html`）

在管理台之外提供 **侧栏式** 的考试专题界面，适合「只负责考务」的人员使用。

- **创建考试**：考试 ID、**考试名称**（展示用）、考生数量、每人配额；可上传试卷文件，选项 **覆盖已存在文件**。
- **操作指引**：文案明确三步：**创建考试 → 上传试卷池 → 随机分卷**。
- **生成结果表**：创建成功后展示 **6 位纯数字** 考生账号列表，支持 **复制全部密码**、**导出 CSV**（含工位号等列，与前端表格一致）。
- **考试历史**：左侧考试列表 + 详情区，可查看场次、刷新、进入分卷/清理等（与嵌入版能力互补）。
- **用户管理**：考务场景下快速调整与考试相关的用户权限（权限选项与管理员台一致：`elr` / `elradfmw` / `elradfmwMT`）。

### 2.5 用户文件浏览器（`/explorer`，`user_explorer.html`）

面向 **终端用户** 的 Web 文件管理界面，与 FTP **共用同一套用户与主目录**，权限受 `perms` 与后端 API 校验约束。

#### 2.5.1 登录与个性化

- 登录页提供返回门户链接；支持 **记住密码**。
- 顶部展示欢迎语与当前用户标识。

#### 2.5.2 考试相关提示（考生友好）

- **已分卷**：显示醒目的 **「我的试卷」横幅**，试卷文件名、路径说明，一键 **查看试卷**（跳转到卷面所在逻辑路径）。
- **未分卷 / 等待中**：**考试信息** 提示条（黄色边条），展示考务状态文案，减少「找不到卷子」的困惑。

#### 2.5.3 文件与目录操作

- **面包屑导航**：从根目录逐级进入子文件夹，可点击返回上级。
- **新建文件夹**、**网格 / 列表视图切换**。
- **搜索**：当前目录下按文件名 **前端过滤**（即时隐藏不匹配项）。
- **拖拽上传区**：大块拖放区域；顶部 **上传** 按钮打开弹窗。
- **上传弹窗**：支持 **多文件**、**整个文件夹**（`webkitdirectory`）；**进度条** 与百分比反馈。
- **文件列表**：文件夹与文件分区图标；支持进入子目录、返回上级。
- **右键菜单**：**下载**、**重命名**、**删除**、**分享**。
- **空状态**：目录无内容时的引导文案。

#### 2.5.4 分享（界面能力说明）

- 右键 **分享** 可设置 **可选密码**、**过期天数**（1–365），并尝试通过 `POST /api/share/create` 生成分享 URL。
- **部署注意**：当前仓库的 `web_server.py` 中 **未包含** `share` 相关路由时，点击「生成链接」会请求失败；界面仍会先展示占位 URL。上线前请确认后端是否已实现分享 API，或暂时勿向用户承诺该功能。

### 2.6 FTP 服务（`ftp_server.py`）

- **账号体系**：与 MySQL `users` 表统一；**拒绝匿名登录**。
- **主目录**：读取库中 `home_dir`，在 Windows 下将 `/ftp_root/...` 规范到项目目录，避免解析到错误盘符。
- **限速**：全局限速配置 + 用户级限速（与管理台字段对应）。
- **配额**：上传时校验 **已用空间 + 配额**。
- **被动模式**：可配置被动端口范围；跨 NAT 时需配合 `masquerade_address` 与防火墙。
- **日志**：登录、传输等行为写入 **数据库日志表** 与 **文件日志**（`ftp_server.log`），供管理台与审计使用。
- **断点续传**：基于 pyftpdlib 能力，适合大文件上传场景（具体客户端需支持 REST）。

### 2.7 与「在线答题」的关系

本项目的「考试」主要指 **试卷文件的分发、考生目录与 FTP 交卷路径管理**，**不是**内置浏览器里做题判分的机考引擎。考生通常：**用浏览器或 FTP 拿到试卷文件 → 本地作答 → 按监考要求上传答卷到个人目录**。

## 3. 运行前提与环境

### 3.1 软件要求

| 组件 | 说明 |
|------|------|
| Python | 建议 3.8+ |
| MySQL | 8.0+，且 **仅允许本机连接**（`localhost` / `127.0.0.1` / `::1`），与 `config.py` 中的校验一致 |
| 操作系统 | Windows / Linux / macOS 均可；FTP 主目录路径在 Windows 下已做专门解析，避免盘符根目录误解析 |

### 3.2 Python 依赖

主要依赖见 `requirements.txt`（Flask、SQLAlchemy、pyftpdlib、pymysql、bcrypt、psutil 等）。

若使用项目根目录的 `.env` 加载配置，代码中使用了 `python-dotenv`。若环境中未安装，可执行：

```bash
pip install python-dotenv
```

与 `pip install -r requirements.txt` 一并安装即可。

### 3.3 数据库说明（重要）

当前版本的 `config.py` **仅支持本地 MySQL**，连接串中的主机必须是本机；非 MySQL 或非本机会在启动时抛出错误。这与部分旧版文档中「自动回退 SQLite」的描述不一致，部署时请以 **本仓库实际代码** 为准。

首次部署建议：

1. 创建数据库并执行 `scripts/init_mysql.sql`（含默认库名 `ftpdb`、初始表、默认 `admin` / `guest` 用户等）。
2. 配置 `.env` 或环境变量中的 `MYSQL_URL`（或 `MYSQL_USER` / `MYSQL_PASSWORD` / `MYSQL_HOST` / `MYSQL_PORT` / `MYSQL_DB`）。
3. 运行 `python main.py`，程序会调用 `db_helper.init_db_tables()` 创建或补齐 ORM 表结构，并对 MySQL 做 **幂等** 的字段兼容与外键修复。

历史数据若曾在 SQLite 中，可参考 `scripts/migrate_to_mysql.py` 迁移到 MySQL（脚本面向旧路径 `data/ftp_server.db`，使用前需核对路径与版本）。

## 6. 核心架构

### 6.1 进程与线程

- **FTP**：由 `FTPServerController` 在 `main.py` 中启动。
- **Web**：`Flask app.run(threaded=True)` 在守护线程中运行；`main.py` 将 `ftp_server` 的控制器注入 `web_server.controller`，供管理接口启停 FTP。
- **认证**：Web 侧支持 Flask `session`、带作用域的 Cookie（管理员与文件浏览器隔离）、`auth_session` 查询参数及 HMAC 签名 token，避免管理端与资源浏览器互相挤掉登录状态。

### 6.2 FTP 子系统要点

- 使用 **pyftpdlib**，自定义 **MySQL 认证**（`db_helper.authenticate_user`），拒绝匿名登录。
- 用户主目录来自数据库 `home_dir` 字段，并在 Windows 上将 `/ftp_root/...` 等形式 **规范化为项目下的物理路径**。
- 支持 **限速**（`ThrottledDTPHandler`）、**配额**、**被动端口范围**（默认约 60000–60099，可在配置中调整）。
- 操作写入 **FTP 日志表**（及文件日志），便于管理端审计。

### 6.3 Web 子系统要点

- 管理仪表盘可聚合 **连接数、用户、磁盘、日志** 等（具体指标以实现为准），部分接口带 **短缓存** 减轻压力。
- **SSE**（如 `/api/stream`）用于实时或准实时推送（日志流等）。
- 文件类 API：`/api/files`、`/api/upload`、`/api/download`、`/api/delete`、`/api/mkdir`、`/api/rename` 等，均与用户身份及主目录绑定。

### 6.4 考试子系统要点

考试逻辑注册在 Blueprint `exam_bp`，URL 前缀为 **`/api/exam`**，与页面 `exam_admin.html`、管理台嵌入页、目录 `ftp_root/exam_paper_pool`、`ftp_root/students` 协同。**界面级操作步骤与按钮说明见上文第 2.3.5、2.4 节。**

主要能力（与路由对应）包括：

- 批量创建考试用户、上传试卷文件、按考试 ID 管理试卷池。
- **随机分发**：为不同考生分配不同试卷文件，并记录到 `exam_students` 等表。
- 列表、历史、按考试导出学生与账号信息、删除单个学生或整考试数据、删除试卷等。
- 分卷后可通过 **FTP 协议** 或 **本地文件复制** 将试卷送达考生目录（实现中优先稳定的是文件系统复制路径，FTP 上传为可选实现）。

数据表核心实体（见 `models.py`）：

- **`exams`**：考试元数据（名称、描述、每生配额等）。
- **`exam_papers`**：某 `exam_id` 下的多份试卷文件记录。
- **`exam_students`**：考生与 `users` 关联、分到的试卷、工位号、明文密码（用于导出）、学生目录等。

## 7. 数据模型摘要

| 表 / 模型 | 作用 |
|-----------|------|
| `users` | Web/FTP 用户，bcrypt 密码、`role`、`perms`（FTP 权限字）、主目录、配额与已用量 |
| `ftp_config` | 全局 FTP 参数单行配置 |
| `ftp_logs` | FTP 行为审计日志 |
| `exams` | 考试定义 |
| `exam_papers` | 试卷池条目 |
| `exam_students` | 考生分卷与账号导出相关字段 |

`db_helper._ensure_mysql_schema_compatibility()` 会在 MySQL 上自动补齐历史版本缺失列，并修正部分外键删除规则（如 `user_id` 级联删除）。

## 8. API 速览

以下为常用接口分类，详细请求体与权限以代码为准。

**认证**

- `POST /api/auth/login`、`POST /api/auth/logout`、`GET /api/auth/me`

**文件（需登录）**

- `GET /api/files`、`GET /api/download`、`POST /api/upload`、`POST /api/delete`、`POST /api/mkdir`、`POST /api/rename`

**管理与监控（多需管理员）**

- 用户：`GET/POST /api/admin/users`，`PUT/DELETE /api/admin/users/<id>`
- 概览与健康：`GET /api/admin/overview`、`GET /api/health`、`GET /api/status`
- FTP 服务：`POST /api/server/start|stop|restart`，`GET/POST /api/server/config` 或 `/api/admin/config`
- 日志：`GET /api/admin/logs`，流式相关 `/api/stream`、`/api/admin/logs/stream`

**考试（前缀 `/api/exam`）**

## 9. 安全与运维建议

1. **默认口令与 `SECRET_KEY`**：部署后立即修改。
2. **网络**：仅在内网或 VPN 开放 FTP 与控制端口；公网需配合防火墙、反向代理与 TLS（FTP over TLS 需额外开发/配置，当前以明文 FTP 为主）。
3. **被动模式**：若客户端跨 NAT，需配置 `masquerade_address` 与被动端口范围，并在防火墙上放行。
4. **备份**：定期备份 MySQL 与 `ftp_root` 物理目录。
5. **权限**：遵循最小权限原则配置数据库账号；考试导出含 **明文密码** 字段时需妥善保管导出文件。

## 10. 常见问题（与代码行为对齐）

- **无法启动并提示仅支持本地 MySQL**：检查 `MYSQL_URL` 主机是否为 `localhost` / `127.0.0.1` / `::1`，且使用 MySQL 驱动连接串。
- **端口占用**：使用 `--ftp-port` / `--web-port` 更换端口。
- **Windows 下主目录错乱**：代码已对 `/ftp_root/...` 做专门解析；若仍异常，检查数据库中 `home_dir` 写法是否为约定的 Unix 风格路径。
- **`/explorer/login.html` 与模板**：若路由引用模板名在仓库中不存在，访问该路径可能报错；正常使用 `/login` 与 `/explorer` 即可。
